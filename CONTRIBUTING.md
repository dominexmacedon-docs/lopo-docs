# Contributing to Lopo

Thank you for your interest in contributing to **Lopo**! 
We welcome contributions from the community to help improve the language, editor, and documentation.

---

##  How to Contribute

### Reporting Issues
- Use the [Issues](../../issues) tab to report bugs, request features, or ask questions.
- Before opening a new issue, please check if one already exists.
- Provide clear steps to reproduce bugs and include screenshots or code snippets when possible.

### Suggesting Enhancements
- Open an issue describing your idea and why it would benefit Lopo.
- Be specific about the use case and potential implementation.

### Submitting Code
1. **Fork** the repository.
2. **Clone** your fork locally.
3. Create a new branch:
   ```bash
   git checkout -b feature/my-new-feature
   ```
4. Make your changes with clear commit messages.
5. Push your branch to your fork:
   ```bash
   git push origin feature/my-new-feature
   ```
6. Open a **Pull Request** against the `main` branch.

---

##  Coding Guidelines
- Keep code clean, readable, and consistent with existing style.
- Add comments for complex logic.
- Write small, focused commits.
- Ensure your code runs in Node.js without breaking existing functionality.

---

##  Documentation
- Update `README.md` or create new docs if your changes introduce new features.
- Add examples in the `examples/` directory when relevant.
- Keep explanations concise and beginner-friendly.

---

##  Testing
- Test your changes locally before submitting.
- If possible, add test cases to cover new functionality.
- Ensure existing examples still run correctly.

---

##  Community Standards
- Be respectful and constructive in discussions.
- Follow the `[Looks like the result wasn't safe to show. Let's switch things up and try something else!]`.
- Collaboration and learning are encouraged - beginners are welcome!

---

##  Releases
- Maintainers will handle versioning and publishing releases.
- Contributors may suggest release notes in the `CHANGELOG.md`.


## Recognition
All contributors will be credited in the repository.  
Your effort helps make Lopo better for everyone!

**Happy coding, and thank you for helping build Lopo!**
